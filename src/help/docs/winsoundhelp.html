Native Sound Card Access

Overview

This plugin module allows PAMGuard to use low level sound card functions, bypassing JavaSound. On some platforms, this allows higher sample rates than are possible with the 'normal' Sound Cards), which use the standard JavaSound libraries.

To use the plugin you will need a native library to access the sound card on the system you are using. On Windows, this is winmmsound.dll, which should be in the lib64 folder of the PAMGuard installation folder.

For other operating systems, you may need to build the library yourself using / modifying code at https://github.com/douggillespie/WindowsSoundJNA.

The plugin has been successfully tested with two channel Cosmos E1DA sound cards at sample rates of 384kHz.
